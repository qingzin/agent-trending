# Release Checklist

在正式发布到 GitHub 前，建议至少检查下面这些事项：

## 必做项

- 确认仓库名、简介和标签
- 确认 `README.md` 首页展示正常
- 确认所有 Markdown 文件为 UTF-8 编码
- 确认不存在个人隐私、密钥、内部备注
- 确认 License 与你的开放范围一致
- 确认目录命名已经稳定，不准备短期大改

## 建议项

- 给仓库补一个封面图或社交预览图
- 在 README 中补上作者署名和联系方式
- 给重点文档补目录或跳转链接
- 增加一个 `CITATION` 或引用说明
- 首次发布打一个 tag，例如 `v0.1.0`

## 上线后可继续做

- 增加英文摘要
- 增加按主题的总索引页
- 增加新增月份的更新日志
- 为重要报告补参考来源列表
